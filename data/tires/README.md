# Reference tyres

Citation-backed `.tyr` datasets (the MF6.1 force model, §7.1) that ship with outlap as reference
and validation data. Coefficient **values** are facts transcribed from published sources with an
exact citation in each dataset's `provenance` block and `README.md`; we never redistribute the
source documents themselves (books/papers stay out of the repo — see the root `.gitignore`).

These live here as data (CC-BY-SA-4.0), **not** in the crate test fixtures — those stay synthetic
and minimal (`crates/outlap-schema/tests/fixtures/tyr/`), per the working agreement.

## Datasets

| Directory | Tyre | Source | Notes |
|---|---|---|---|
| [`pacejka_2006_205_60r15/`](pacejka_2006_205_60r15/) | 205/60R15 91V passenger car | Pacejka, *Tyre and Vehicle Dynamics*, 2nd ed. (2006), Table A3.1 | The book's worked-example car tyre; doubles as the MF6.1 validation tyre. 2nd-edition set: no inflation-pressure (`PP*`) terms; `Mx ≡ 0`; `My` via `qsy1`. |

The Perantoni & Limebeer F1 tyre and the Roborace MF5.2 tyre join in a later milestone increment.

## Blocks that aren't published

`Tyr` requires `thermal` and `wear` blocks, but those models are M5 and the published sources give
only force/moment coefficients. Each dataset therefore carries **synthetic, clearly-labelled**
thermal/wear placeholders (physically-plausible for the tyre class), noted in `provenance.source`
and in a `# SYNTHETIC` comment on the block. `synthetic: false` on a dataset means the load-bearing
force/moment coefficients are the published measured set — the placeholder blocks are the documented
exception.

## Validation

Every `data/**/*.tyr.yaml` is checked against the published JSON Schemas by
`python -m outlap.schemas --check` (CI). The Pacejka tyre additionally has a Rust integration test
(`crates/outlap-tire/tests/reference.rs`) asserting it loads warning-free and evaluates to
physically-sane forces (grip band, sign conventions, finiteness). The tight `≤ 0.5%` cross-check
against golden CSVs (generated by the teasit Magic-Formula library, outputs-as-data only) is
described in [`tools/goldens/`](../../tools/goldens/).
